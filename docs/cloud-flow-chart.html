1. Big picture

Two products share the same ElizaCloud backend packages. Milady was the desktop shell (Electrobun + agent runtime). Detour Cloud is the reseller dashboard that vendors cloud-api plumbing and bills via $DTOUR + Convex.

3. cloud-api request lifecycle

Every HTTP hit to api.elizacloud.ai follows this stack (see packages/cloud-api/src/index.ts → bootstrap-app.ts).

sequenceDiagram participant C as Client participant W as Cloudflare Worker participant H as Hono bootstrap-app participant A as authMiddleware participant R as route handler participant S as cloud-shared service participant D as Postgres / R2 C->>W: HTTPS request W->>H: fetch(request, env, ctx) H->>H: setRuntimeR2Bucket + runWithCloudBindingsAsync H->>H: requestId, CORS, secureHeaders H->>A: authMiddleware (public paths skip) alt Bearer eliza_* or API key A->>R: pass through (handler validates org) else Steward cookie/JWT A->>A: getCurrentUser A->>R: set user on context end R->>S: business logic S->>D: read/write S-->>R: result R-->>C: JSON / stream

Route discovery

packages/cloud-api/<path>/route.ts maps to /api/<path> (Next.js App Router conventions). Run bun run codegen in cloud-api after adding routes.
